# Haalan 电商平台 - 项目技术文档

## 项目概述

Haalan 是一个基于微服务架构的电商平台，采用Spring Cloud Alibaba技术栈构建。该项目是一个完整的电商解决方案，包含了用户管理、商品管理、订单处理、秒杀系统、搜索功能等核心模块。

项目前端已写好很大一部分,尚不完善,如需访问请联系作者开放;
## 技术栈

- **基础框架**: Spring Boot 2.7.12, Spring Cloud 2021.0.3
- **微服务组件**: Spring Cloud Alibaba (Nacos, Sentinel, Seata)
- **数据库**: MySQL 8.0
- **ORM框架**: MyBatis-Plus 3.5.5
- **缓存**: Redis, ConcurrentHashMap本地缓存
- **消息队列**: RabbitMQ
- **搜索引擎**: Elasticsearch 7.12.1
- **API网关**: Spring Cloud Gateway
- **安全认证**: JWT
- **开发工具**: Maven, Lombok, Hutool

## 项目架构

该项目采用微服务架构模式，主要包括以下模块：

### 1. haalan-gateway (网关服务)

- **端口**: 8080
- **功能**: 统一入口、身份认证、动态路由
- **关键技术**: Spring Cloud Gateway, JWT, Redis
- **特性**:
    - 动态路由配置
    - 全局身份验证
    - 请求过滤与限流

### 2. haalan-common (公共组件)

- **功能**: 提供通用工具类、配置类、异常处理等
- **包含模块**:
    - 统一响应格式 (R类)
    - 分页工具 (PageDTO)
    - 异常处理 (BizIllegalException)
    - 工具类 (JWT工具、Redis工具等)
    - 拦截器 (UserContext)

### 3. haalan-api (服务接口)

- **功能**: 定义各服务之间的远程调用接口
- **关键技术**: OpenFeign
- **特性**: 服务间通信、负载均衡

### 4. user-service (用户服务)

- **功能**: 用户注册、登录、个人信息管理
- **关键技术**: JWT、BCrypt密码加密、Redis
- **特性**:
    - 用户身份认证
    - 密码安全加密
    - 会话管理

### 5. item-service (商品服务)

- **功能**: 商品信息管理、库存管理、商品上下架
- **关键技术**: MyBatis-Plus、Elasticsearch
- **特性**:
    - 商品信息维护
    - 库存管理
    - 商品数据同步到ES

### 6. order-service (订单服务)

- **功能**: 订单创建、支付处理、订单状态管理
- **关键技术**: Seata分布式事务、RabbitMQ消息队列
- **特性**:
    - 分布式事务管理
    - 支付集成（支付宝）
    - 订单状态流转

### 7. seckill-service (秒杀服务)

- **功能**: 秒杀活动管理、高并发秒杀处理
- **关键技术**: Redis Lua脚本、分布式锁、本地缓存
- **特性**:
    - 高并发秒杀处理
    - 库存预热机制
    - 防刷机制
    - 令牌桶算法
    - Redis+本地双重缓存

### 8. search-service (搜索服务)

- **功能**: 商品搜索、全文检索
- **关键技术**: Elasticsearch
- **特性**:
    - 全文搜索
    - 分类筛选
    - 排序功能

## 核心功能模块详解

### 秒杀系统设计

秒杀系统是本项目的核心亮点之一，采用多层防护设计：

#### 1. 预热机制

- 将秒杀商品信息预加载到Redis
- 预热库存数据，减少数据库压力
- 支持活动级别的预热

#### 2. 本地缓存优化

- 使用JVM本地缓存做第一层过滤
- 售罄状态本地标记，快速失败
- 减少Redis访问压力

#### 3. Redis + Lua脚本

- 使用Lua脚本保证原子性操作
- 防止超卖现象
- 实现用户限购逻辑

#### 4. 令牌机制

- 生成一次性令牌，防止重复提交
- 令牌时效性控制
- 防止恶意刷单

#### 5. 消息队列异步处理

- 秒杀成功后异步创建订单
- 解耦秒杀核心流程
- 提高系统响应速度

### 分布式事务处理

使用Seata实现分布式事务，确保跨服务的数据一致性：

- 订单创建涉及多个服务的协调
- 库存扣减的事务保障
- 支付状态的一致性

### 搜索功能

集成Elasticsearch提供高效的商品搜索：

- 商品信息同步到ES
- 全文检索功能
- 多维度筛选和排序

### 网关认证

网关层统一处理认证和授权：

- JWT Token验证
- 路径白名单配置
- 用户信息透传

## 数据库设计

项目采用分库分表策略：

- 用户、商品、订单等核心数据分离
- 秒杀订单分表存储
- 合理的索引设计提升查询性能

## 部署架构

- 配置中心：Nacos
- 服务监控：健康检查
- 负载均衡：Ribbon/LoadBalancer
